# HTML and CSS course project progression

Project theme: **Brew & Bean** — a simple coffee shop page that grows lesson by lesson.

## Recommended lesson order and how each lesson contributes

1. **01 HTML introduction**  
   Start with a plain `index.html` file and basic `p`, `h1`, and `h2` tags.

2. **02 Viewing output**  
   Keep the page simple while learners practise opening the file in Chrome, using live preview, and switching browser modes.

3. **03 HTML document structure**  
   Turn the plain page into a real HTML document with `DOCTYPE`, `html`, `head`, and `body`, then add semantic structure, an image, and lorem ipsum text.

4. **04 CSS introduction**  
   Link `style.css` and style the page with only the CSS ideas introduced here: selectors by tag plus `background`, `color`, `font-size`, and `font-family`.

5. **05 CSS selectors**  
   Add `class` and `id` attributes so the project can target specific parts of the page.

6. **06 CSS colours**  
   Apply a more complete colour palette to turn the page into a coffee-shop design.

7. **07 CSS text**  
   Improve the typography with heading and paragraph font choices and sizes.

8. **08 Block level elements and sizing**  
   Add widths and heights using `vw`, `vh`, `em`, and `px` ideas from the lesson.

9. **10 Borders, margins, padding, horizontal centering**  
   Add spacing, borders, rounded corners, and centered panels. This is the first lesson where margin and padding are used.

10. **11 Positioning with fixed, absolute, relative**  
    Fix the header, make the hero area relative, and place a small note absolutely inside it.

11. **12 Scrolling**  
    Turn the middle area into the scroll container so the content scrolls while the header and footer stay visible.

12. **13 CSS flexbox**  
    Use flexbox for the nav, hero layout, and menu rows, including vertical centering.

13. **14 CSS hover**  
    Add hover states to navigation and menu cards.

14. **15 Build simple UI with scrolling**  
    Finish the mini interface by adding more content to the scrollable layout.

## Important curriculum rule used in these files

Each folder contains the **end-of-lesson source code** for that lesson, and that code becomes the starting point for the next lesson.

The examples are intentionally conservative:
- no classes or ids before the selectors lesson
- no margin, padding, border, or border-radius before lesson 10
- no positioning before lesson 11
- no overflow scrolling before lesson 12
- no flexbox before lesson 13
- no hover before lesson 14

That keeps each version aligned with what has already been taught.
