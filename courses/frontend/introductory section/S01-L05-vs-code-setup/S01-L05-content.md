# Lesson — Setting Up VS Code

---

# Slide 1

**Title**

VS Code Setup

**Bullet Points**

• Install a few recommended extensions
• Enable some helpful editor settings
• Match the course development environment

---

### Narrative

In this lesson, we’ll quickly set up **Visual Studio Code**, which will be the editor used throughout the course.

You don’t need to match my setup exactly, but installing a few extensions and enabling a couple of settings will help ensure that your editor behaves the same way mine does in the lessons.

This will make it easier to follow along as we build the project together.

---

# Slide 2

**Title**

Recommended Extensions

**Bullet Points**

• ES7+ React/Redux snippets
• Live Preview
• Prettier
• TabOut

---

### Narrative

First, we’ll install a small set of extensions.

These extensions provide useful shortcuts, help preview web pages, and automatically format our code so everything stays clean and consistent.

If you'd like your environment to match the one used in the course, I recommend installing these now.

[ Go to VS Code and demonstrate installing the extensions ]

Open the **Extensions panel**, search for each extension, and install them one by one.

Once those are installed, we’ll configure a few editor settings.

---

# Slide 3

**Title**

Recommended Settings

**Bullet Points**

• Format code automatically on save
• Use Prettier as the default formatter
• Enable linked editing
• Configure localhost browser behaviour

---

### Narrative

Next, we’ll enable a few editor settings used throughout the course.

These settings enable automatic formatting with Prettier, improve editing when working with HTML and JSX, and control how localhost links open while we run our applications.

[ Go to VS Code and demonstrate editing the settings JSON ]

Open the **Settings JSON file**, and add the settings used in this course.

Once those are in place, your editor should behave the same way mine does in the lessons.
